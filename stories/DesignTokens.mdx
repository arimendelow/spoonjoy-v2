# Design Tokens

The fundamental building blocks of Spoonjoy's visual language. These are the atoms from which all UI molecules are formed. Or, if chemistry metaphors aren't your thing: the Lego bricks of our design system.

Design tokens are the single source of truth for colors, spacing, typography, and other visual properties. Change them here, and watch the entire UI update in horrifying synchronicity.

### Tailwind Default Colors

We also have access to the full Tailwind color palette. These are the factory-installed colors that ship with every Tailwind project. Here are the highlights:

| Color | Classes | Use Case |
|-------|---------|----------|
| Slate | `bg-slate-*`, `text-slate-*` | Neutral grays with a cool undertone |
| Gray | `bg-gray-*`, `text-gray-*` | True neutral grays |
| Zinc | `bg-zinc-*`, `text-zinc-*` | Slightly warmer grays |
| Red | `bg-red-*`, `text-red-*` | Errors, destructive actions |
| Orange | `bg-orange-*`, `text-orange-*` | Warnings, attention |
| Yellow | `bg-yellow-*`, `text-yellow-*` | Caution, highlights |
| Green | `bg-green-*`, `text-green-*` | Success, positive actions |
| Blue | `bg-blue-*`, `text-blue-*` | Links, information |
| Indigo | `bg-indigo-*`, `text-indigo-*` | Accents, special states |
| Purple | `bg-purple-*`, `text-purple-*` | Premium, creative |
| Pink | `bg-pink-*`, `text-pink-*` | Fun, playful |

Each color comes in shades from `50` (lightest) to `950` (darkest). For example: `bg-blue-500`, `text-slate-900`.


## Breakpoints

Responsive design boundaries. Where layouts go to transform.

| Breakpoint | Min Width | CSS | Example |
|------------|-----------|-----|---------|
| `sm` | 640px | `@media (min-width: 640px)` | `sm:flex` |
| `md` | 768px | `@media (min-width: 768px)` | `md:grid-cols-2` |
| `lg` | 1024px | `@media (min-width: 1024px)` | `lg:px-8` |
| `xl` | 1280px | `@media (min-width: 1280px)` | `xl:max-w-6xl` |
| `2xl` | 1536px | `@media (min-width: 1536px)` | `2xl:grid-cols-4` |

### Mobile-First Approach

Tailwind uses mobile-first breakpoints. Unprefixed classes apply to all sizes, prefixed classes apply at that breakpoint and up.

```html
<!-- Stack on mobile, side-by-side on tablet and up -->
<div class="flex flex-col md:flex-row">
  <div>First</div>
  <div>Second</div>
</div>

<!-- 1 column mobile, 2 on tablet, 3 on desktop, 4 on large -->
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4">
  <!-- items -->
</div>
```

## The Philosophy

These tokens exist so you never have to argue about whether 14px or 15px is the "right" padding. The scale has spoken. Use `p-3.5` or `p-4` and move on with your life.

Consistency is the goal. When everything follows the same scale, the UI feels cohesive even when built by different developers at different times. That's the magic of design tokens.

Now go forth and build beautiful things.
